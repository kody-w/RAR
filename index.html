RAR Quality Seal Program

Every agent in the registry carries a quality tier. Like Nintendo's Seal of Quality, the RAR Verified seal means an agent meets a rigorous standard of reliability, security, and usefulness.

⚖ FRONTIER Preview Tier

For agents pushing the edge. Ship early, iterate fast, break new ground.
  • Valid __manifest__ with all required fields
  • Follows the single-file agent.py pattern
  • Parseable by AST (no syntax errors)
  • No hardcoded secrets, API keys, or credentials
  • Clearly marked as frontier in the registry
  • Pushing the edge — may be evolving rapidly
  • Can be promoted to Community when stable

COMMUNITY Open Tier

The starting line. Submit your agent and join the community.
  • Valid __manifest__ with all required fields
  • Follows the single-file agent.py pattern
  • Parseable by AST (no syntax errors)
  • Passes automated contract tests (test_agent_contract.py)
  • No hardcoded secrets, API keys, or credentials
  • No PII (scanned by pre-commit scrubber)

RAR VERIFIED Quality Seal

Reviewed by real humans. Tested in production. The ones you can count on.
  • All Community tier requirements
  • Extends BasicAgent properly with correct perform() signature
  • Meaningful perform() implementation (not just a stub)
  • Docstring serves as useful documentation
  • Proper error handling — graceful failures, no bare except:
  • Environment variables declared in requires_env are actually used
  • Dependencies declared in dependencies are accurate
  • Minimum 3 community upvotes
  • At least 1 review with rating >= 3 stars
  • Code reviewed by a registry maintainer
  • No obfuscated code, no minification, no encoded payloads
  • Clean separation of concerns — single responsibility

OFFICIAL Core Agents

Built and maintained by the core team. The bedrock of the ecosystem.
  • All Verified tier requirements
  • Maintained by core team with guaranteed SLA
  • Backwards-compatible across RAPP versions
  • Integration tested against live infrastructure
  • Versioned with semantic versioning (breaking changes = major bump)
  • Included in the RAPP release validation suite

How to Get Verified

  1. Submit your agent through the Submit tab
  2. Accumulate at least 3 community upvotes and 1 review
  3. Open an issue titled [RAR] verify: @your-name/agent-slug
  4. A maintainer will audit the code against the Verified checklist above
  5. Once approved, the agent's quality_tier is upgraded to "verified"

2. Push streamed agent bytecode to the Brainstem's agent loader
3. Route deck GUIDs to the Brainstem's perform() loop
4. Receive execution results back for display in the steering panel

Ollama — Local-First AI for Brainstem

Replace cloud AI providers with on-device inference. Ollama runs Gemma 4 locally — the Brainstem routes agent reasoning through it. Zero API keys. Zero data leaves your machine.
Launch command:
ollama launch brainstem --model gemma4
This starts Ollama, pulls the model if needed, and launches the Brainstem runtime backed by local inference.

How it works

Stream — Fetches raw .py from GitHub into a JS Map. No file write. Purely in-memory.
Hot-Load — Manifest is parsed, agent appears in browse/cards/search instantly.
Deck Router — Stream an entire deck. Each agent gets a GUID. The deck topology defines the routing group.
Steer — Route state through GUID-mapped agents. The same loop that fires perform() can be steered by swapping which agents are live.
Eject — Remove from memory. Gone. No trace on disk.
Brainstem — Connect to a local RAPP Brainstem. Push agent bytecode for on-device execution. The browser steers, the Brainstem performs.
Ollama — True local-first AI. Run Gemma 4 on-device via Ollama. The Brainstem reasons with local models — no cloud, no API key. ollama launch brainstem --model gemma4
CommunityRAPP — Route to a hosted instance. Same agents, cloud execution surface.
Flick — Pick a card, attach a task, flick it to a runtime. The card carries bytecode, the flick carries intent. Multiple flicks in parallel = multiple agents with their own data streams.
Encrypt — E2E encryption via AES-256-GCM. Generate a key, share via QR code on the card back. From scan to stream to steer — encrypted end-to-end.
Universal — The .py file is the unit. The runtime is a choice. Stream once, route anywhere.

The Registry

The single source of truth for agent discovery. One file = one agent = one card in the registry.

✓
AST-based manifest extraction
Parse __manifest__ from .py files without executing code. Safe, fast, deterministic.
✓
Issues-as-API
Submit, vote, and review agents through GitHub Issues. No server. GitHub Actions processes, validates, commits, closes.
✓
4-tier quality seal
Experimental → Community → Verified → Official. Only community and experimental are self-submittable.
✓
Federation
Clone as template. Run your own store. Submit deltas upstream. The main store signs and distributes.
●
Agent dependency graph
Resolve dependencies declared in manifests. Show the graph. Auto-bundle when downloading.
○
Version diffing
Compare versions of an agent side-by-side. See what changed in perform(), manifest, and dependencies.

The Workbench

Build agents locally, test them, iterate, then publish. The browser is your IDE.

✓
Local-first agent loading
Drag and drop .py files into the browser. Stored in IndexedDB. Persists across sessions. No upload needed.
✓
Starter templates
Blank, Hello World, API Caller, Data Processor. Customize and submit. Unmodified templates are caught and rejected.
✓
Manifest validation
Real-time validation as you type. Name, version, publisher match, tier enforcement — all checked before submit.
●
In-browser code editor
Edit local agents directly in the workbench. Syntax highlighting, manifest preview, one-click submit.
○
Agent testing sandbox
Invoke perform() with test inputs and inspect outputs. Mock env vars. Dry-run before publishing.
○
Fork & remix
Take any registry agent, fork it to your namespace, modify it, and republish. One-click workflow.

Trust & Signing

RAR signs agents like Apple signs apps. The seal means something.

✓
Publisher namespaces
@yourname/ is yours. Enforced at submission. No impersonation. No squatting.
✓
Automated contract tests
Every agent in the registry passes AST parsing, manifest validation, and structural contract checks on every push.
●
Code signing via commit
The main repo is the certificate authority. An agent in the registry = signed by RAR. Git commit hash = signature.
○
Integrity verification
Verify a local .py against the registry hash. Detect tampering. "This card is authentic."
○
Security scanning
Automated scan for hardcoded secrets, network calls in __init__, obfuscated code. Block before merge.
